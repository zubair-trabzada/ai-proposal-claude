# Proposal Pricing — 3-Tier Pricing Builder

Builds Good-Better-Best pricing with ROI math for a service. Invoked by `/proposal pricing <service>`. Output: `PROPOSAL-PRICING-[Service].md`.

Works for **any** service — a web build, automation setup, marketing retainer, consulting, coaching, GEO/SEO, whatever.

## Steps
1. Identify the service and engagement model (one-time project / monthly retainer / value-based / hybrid).
2. Set the price anchor: use the user's own rates first; else value-based (a fraction of the value created); else a market reference. Fallback AI-adjacent bands: web build $1.5K–10K · automation $2K–15K (or $500–5K/mo) · marketing $1K–5K (or $2K–10K/mo) · GEO/SEO $500–2.5K (or $1.5K–10K/mo) · consulting $1.5K–10K.
3. Adjust for client size, location (HCOL ×1.05–1.15), urgency, buyer sophistication.
4. Build 3 tiers with aspirational names:
   - **Starter/Launch** — foot in the door, smallest viable scope, low end
   - **Growth/Core ★ RECOMMENDED** — the complete solution, midpoint, the target
   - **Scale/Partner** — premium anchor, expanded scope, high end
5. ROI math per tier: investment → expected return → breakeven → 12-mo ROI multiple. Conservative must stay positive.

## Rules
Top tier anchors; middle is RECOMMENDED; bottom never too complete. Never a price without ROI context. If an audit score exists, frame price against the point gap in revenue terms.

## Output
A pricing comparison table (feature rows across 3 tiers), ROI math block per tier, and a one-line recommendation on which tier to push given the client's budget.
