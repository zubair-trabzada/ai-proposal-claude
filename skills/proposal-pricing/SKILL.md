# Proposal Pricing — 3-Tier Pricing Builder

Builds Good-Better-Best pricing with ROI math for a service. Invoked by `/proposal pricing <service>`. Output: `PROPOSAL-PRICING-[Service].md`.

## Steps
1. Identify service + market band:
   GEO $500–2K once / $1.5K–8K mo · SEO $750–2.5K / $2K–10K mo · Reputation $500–1.5K / $1K–5K mo · Restaurant $500–1.5K / $1.5K–6K mo · Recruiting $1.5K–5K / $3K–12K mo.
2. Adjust for client size, location (HCOL ×1.05–1.15), urgency.
3. Build 3 tiers with aspirational names:
   - **Starter/Audit** — foot in the door, one-time, low end
   - **Growth ★ RECOMMENDED** — audit + monthly, midpoint, the target
   - **Dominate** — premium anchor, high end
4. ROI math per tier: monthly investment → expected return → breakeven → 12-mo ROI multiple. Conservative must stay positive.

## Rules
Top tier anchors; middle is RECOMMENDED; bottom never too complete. Never a price without ROI context. If an audit score exists, frame price against the point gap in revenue terms.

## Output
A pricing comparison table (feature rows across 3 tiers), ROI math block per tier, and a one-line recommendation on which tier to push given the client's budget.
