# Proposal Follow-Up — Post-Send Email Sequence

Generates the follow-up sequence that turns a sent proposal into a signed deal. Invoked by `/proposal followup <client>`. Output: `PROPOSAL-FOLLOWUP-[Client].md`.

## Sequence (4 emails, all ready to send)
- **Day 0 — Delivery:** attach proposal, highlight the 2 sections to read first, propose a specific walkthrough time. 50-70 words.
- **Day 2 — Walkthrough offer:** ask if reviewed, pre-answer the likeliest question, give two time options. 40-60 words.
- **Day 5 — Value-add:** share one genuine insight, no proposal mention. 50-70 words.
- **Day 9 — Soft close:** honest and direct, leave the door open, one factual urgency point ("Should I close this out?"). 50-70 words.

## Rules
Never guilt-trip or threaten. Each email: one CTA, plain text, peer-to-peer tone, no jargon. Subject lines 4-7 words. End the breakup positively.
