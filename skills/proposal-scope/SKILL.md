# Proposal Scope — Scope & Deliverables Builder

Defines deliverables, a phased plan, and timeline for a service. Invoked by `/proposal scope <service>`. Output: `PROPOSAL-SCOPE-[Service].md`.

## Steps
1. Read any audit report; map each deliverable to a real problem.
2. Build 3 phases: **Foundation** (audit + quick wins), **Growth** (ongoing optimization/content/monitoring), **Scale** (compounding plays + reporting). Each: objective, activities→deliverables, milestone, client involvement.
3. Define precisely: deliverables (with quantity/frequency), meeting cadence, response SLAs, tools included, **≥3 exclusions**, client responsibilities.
4. Timeline: week-by-week table + key dates with realistic buffer.

## Rules
Deliverables must be concrete and measurable, never vague. Always include exclusions to prevent scope creep. Phases build on each other.
